# Roadmap

## Phase 1 (0–6 months): Architecture + Alpha
- Define modular architecture for RMF workflow automation
- Draft SSP/POA&M templates and data models
- Define control mapping approach for NIST SP 800-53
- Publish initial whitepaper (v1)
- Alpha release of documentation + schemas

## Phase 2 (6–12 months): DevSecOps + Zero Trust Modules
- DevSecOps integration model (CI/CD telemetry ingestion, scanners, evidence capture)
- Zero Trust alignment module (identity-centric controls, segmentation mapping, monitoring)
- Add sample workflows for continuous monitoring
- Beta whitepaper (v2)

## Phase 3 (12–24 months): Public Beta + Community Growth
- Expand cross-framework mappings (CMMC, FedRAMP where applicable)
- Add reference implementations (optional, contributor-led)
- Community workshops/webinars
- Pilot collaboration with regulated organizations

## Phase 4 (18–36 months): Commercialization Pathway
- Managed hosting/support options (U.S.-based)
- Compliance-as-a-Service offering (optional)
- Training modules for workforce development
- Hiring plan dependent on adoption and partnerships
